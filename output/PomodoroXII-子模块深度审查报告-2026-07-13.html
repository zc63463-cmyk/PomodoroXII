System boundaries

四层系统边界

主路径由 Browser 经 API 进入 Storage，Agent 侧通过共享 registry 与 services 复用同一业务核心。同步协议是跨层数据一致性的关键边界。

  1. Browser

    Next.js shell → Axios dual JWT → Dexie → Sync engine

  2. API

    FastAPI routes → Registry/meta → Services → SQLAlchemy

  3. Storage

    meta.db + per-space DB + notes filesystem + FTS

  4. Agent

    FastMCP tools/resources/prompts → shared registry/services

HTTP contract /api/v1

Backend baseline

后端：功能基线健康

pytest 全量套件、Ruff、Python 兼容性与已知漏洞审计均给出健康信号。FastAPI 路由、registry/meta、service 与 SQLAlchemy 形成了可识别的边界，FastMCP 通过共享 registry/services 复用业务核心。

运行时信号

783 个测试通过，1 个预期失败，没有普通失败；当前不是后端套件阻断发布。

数据边界

meta.db、per-space DB、notes filesystem 与 FTS 分担不同持久化责任，发布时需保持迁移与文件存储的一致性。

Frontend baseline

lint、typecheck 与包含 19 条路由的 build 通过，证明 Next.js shell 可编译与生成产物。但测试仍是 541 passed, 1 failed，且 API rewrite 固定到 http://localhost:8000，这两项直接破坏非本地交付的可信度。

已通过

仍阻断

测试沙箱仍有一个失败，部署时 API base 无法按环境建立正确上游。

Risk register

核心发现

当前 NO-GO 由三个独立事实支撑：部署边界断裂、证据快照漂移、前端门禁未全绿。任一项未关闭都不应转为 GO。

P1

API base 被锁定到 localhost

非本地运行时无法依赖这个地址，构建通过不能证明 API 链路可用。

P1

审查快照落后远端 11 个提交

必须在对齐远端后重放核心门禁，才能把本地结论升级为当前交付结论。

P2

前端测试尚有 1 个失败

lint、typecheck 和 build 的绿灯不会抵消测试套件失败，必须隔离并复现沙箱问题。

Delivery decision

交付判定：NO-GO

当前不应发布完整前端产品。 后端的健康基线可作为修复与重验的起点，但发布决策必须同时覆盖前端测试、API 环境绑定和远端对齐。

最小放行条件是：API base 可配置且在目标环境验证、前端测试全绿、对齐远端后重跑后端与前端门禁。GitHub 实时 runs 当前未核对，不可当作绿灯。

Index confidence

索引健康：可用于定位，不代替运行时证据

当前代码边界可以通过路由、registry、services、Dexie 与 sync engine 的引用关系进行定位。本报告不把索引命中数当作发布门禁：索引无法抵消前端失败测试，也无法证明远端最新提交已被本地验证。

因此，索引在本次审查中的信心等级是“边界导航”，结论信心仍以本地命令输出、源码核对和 Git 差异为主。

Ordered remediation

行动顺序

  1. P0 · 解除 API base 锁定。让 rewrite 从部署环境取值，并在非 localhost 目标上运行 API 连通性验证。
  2. P0 · 对齐远端差异。审阅 11 个提交的变化面，消解本地漂移，在新树上重放证据台账。
  3. P1 · 修复测试沙箱。单独复现当前失败用例，确认环境与产品行为边界，然后重跑前端全套。
  4. P1 · 重做发布判定。后端、前端、依赖审计与目标环境连通性同时通过后，再将 NO-GO 重评为 GO。

Method and limits

方法与限制

审查将“命令已验证”、“源码可核对”、“远端差异已核对”与“未验证”分开，不由单一绿灯外推整体可发布性。

  • 运行时证据 以 pytest、Ruff、兼容性检查、依赖审计、前端 tests/lint/typecheck/build 的本地输出为准。
  • 源码边界 按 Browser、API、Storage、Agent 四层跟踪路由、registry/services、持久化与同步协议。
  • Git 证据 本地 65e2382 与远端 1e4f0fc 作为快照边界，behind 11 被视为结论时效性风险。
  • 已知限制 GitHub live runs 未检查，因为 gh 未认证；该缺口明确标记为 unverified。
