## 1. 独立审查结论

独立产品审查最初给出“有条件批准”，提出六项 P0：

1. 待复盘的最低完成条件不明确；
2. “接近上限”与“达到上限”的异常阈值冲突；
3. Pomodoro Space 与任务事实缺少同域约束；
4. Session 结果命令部分成功、重试和对账不完整；
5. Session 运行中 WorkItem 重组后的处理不闭环；
6. 二级完成后活动三级项可能悬挂。

上述 P0 已全部写入总纲并闭环。

## 2. 已补齐规则

### 2.1 待复盘最低条件

仅当 Session 存在已触达三级项，且其成果结果既未确认、也未明确跳过时，才产生待复盘。

以下情况不产生待复盘：

- 无三级计划；
- 全部三级均未触达；
- 用户明确选择“跳过本轮成果复盘”；
- 仅总体推进、Mood 或执行化身留空。

### 2.2 投入阈值

- 低于预计下限：不提示；
- 进入预计区间：仅静态信息；
- 达到或超过预计上限：显著复核并可形成推进异常；
- 不再使用未定义的“接近上限”阈值。

### 2.3 Space 同域约束

Project、WorkItem、Cycle、FocusSession 及跨域关联必须携带同一 `spaceId`。跨 Space 引用、移动、命令与聚合一律拒绝。

Space 归档后业务数据只读保留；删除必须经过 PomodoroXII 空间级显式删除流程，任务子系统不得单独删除或迁移。

### 2.4 命令部分成功与对账

每个三级结果命令使用：

```text
sessionId + workItemId + outcomeRevision
```

作为稳定幂等键。命令分别保存：

```text
pending / succeeded / conflicted / failed
```

单项成功或失败不回滚 Session 和其他成功项。失败可重试；冲突需展示当前 WorkItem 状态并由用户决定；Session 提供对账入口。

### 2.5 运行中结构变化

Session 启动时冻结二级归属和三级计划快照。运行中 WorkItem 移动、删除、取消或换父项，不改写本轮。

结束提交前逐项校验：

- 仍存在且状态兼容：可提交；
- 已删除、取消或父项不再匹配：只保留 Session 历史结果，进入冲突处理；
- 不自动移动、复活或覆盖 WorkItem。

### 2.6 父项完成与活动子项

二级完成时若仍有活动三级项，用户必须选择：

- 取消三级项；
- 移动到其他未完成二级；
- 重新打开原二级后继续；
- 返回不完成二级。

不能留下仍可执行的悬挂三级项。纯历史完成/取消项可留在成果区。

## 3. 批准范围

本次批准以下产品方向：

- 单人 Pomodoro Space 隔离模型；
- 轻量 Project Group → Project → WorkItem；
- 统一 WorkItem 三层父子结构；
- 二级承接预计投入、Session 实际投入与 Cycle 容量；
- 三级作为正式成果拆解，不分配 Session 分钟；
- WorkItemNote 与 Checklist 边界；
- Session 混合拆解入口；
- Orbit 计时中执行窗；
- 执行化身模型；
- Session 结束五组信息；
- 真正阻塞、推进异常、待复盘三分法；
- 达到预计上限才显著复核；
- 四个专题原型 + 统一审查 Hub 的组织方式。
