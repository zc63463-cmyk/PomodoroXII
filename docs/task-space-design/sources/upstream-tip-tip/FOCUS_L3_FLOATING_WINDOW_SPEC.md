# 计时态 L3 聚焦式悬浮思维画布规格 v1.0

> 日期：2026-07-10
>
> 状态：已完成产品对齐，进入 HTML 视觉原型实现
>
> 上游契约：`SESSION_TASK_INTEGRATION_V10.md`、`TOPIC_PROTOTYPE_SPEC.md`

## 1. 产品目标

在 FocusSession 运行期间，让 Orbit 桌宠窗口本体切换为一个安静、可移动、可按需扩展的 L3 思维画布，使用户把注意力放在当前二级 WorkItem、本轮三级成果和当前三级的一段行动备注上，而不是面对缩小版任务管理器。

核心理念：

> 像思维导图一样发散拆解任务，并通过节点聚焦保持工作心流。

## 2. 空间模型

L1、L2、L3 是 Orbit 桌宠窗口本身的三种空间视图，不在 L3 内再放置桌宠头像或独立角色组件。

```text
L1：Project / 全局缩略
  ↓
L2：Project + 一级/二级 WorkItem
  ↓ 开始或返回专注
L3：当前二级 + 本轮三级 + 当前三级 Note
```

- Session 运行中返回 L2 只改变观察层级，不暂停或结束计时；
- L2 保留时间胶囊、当前二级点亮和“一键返回专注 L3”；
- 暂停与结束只由时间胶囊明确触发。

## 3. 窗口形态

- 默认小型横向悬浮窗，可自由拖动；
- 不随内容自动侵占桌面；
- 用户可主动进入扩展画布，再缩回原尺寸；
- 紧凑态内容溢出时使用“还有 N 项 / Note · N”，不自动扩窗；
- 扩展和缩回保持当前三级、完成草稿、Note 编辑内容与窗口位置；
- 窗口不使用系统固定侧栏、底部 Dock 或三栏工作台结构。

## 4. 画布构图

采用中心偏左、向右单向发散：

```text
二级主圆
  ├── 三级圆 A
  ├── 三级圆 B（当前项）── 方形 Note
  ├── 三级圆 C
  └── ＋ 原位新建三级
```

规则：

- 二级主圆是稳定结构锚点；
- 本轮涉及或运行中创建的三级常驻画布；
- 其他同父活动三级折叠为“还有 N 项”入口；
- 已完成/已取消项不进入计时态主画布；
- 三级圆大小统一，Note 使用方形节点；
- 连线轻、细、低对比，不使用粗生命线、粒子流或复杂依赖线。

## 5. 三种观察状态

### 5.1 紧凑全景

- 默认状态；
- 显示二级主节点与本轮全部三级；
- 当前三级点亮；
- Note 仅显示微型数量或摘要标记；
- 回答“这一轮整体在推进什么”。

### 5.2 节点聚焦

单击三级节点触发：

- 设为唯一当前处理项；
- 当前节点及其分支移动到窗口视觉中心；
- 二级与其他三级降低对比，但不完全消失；
- 当前三级展开一段 Note 与快速编辑；
- 浮现“详情”“扩展画布”和独立成果钩；
- 切换当前项不分配或重算三级分钟。

### 5.3 窗口聚焦

用户点击“扩展画布”或右上角放大：

- 窗口主动放大；
- 当前三级和 Note 分支保持视觉中心；
- 展示更多候选三级和 Note 摘要；
- 用户可自由移动；
- 缩回后恢复紧凑尺寸和原上下文。

## 6. 计时呈现

顶部居中使用悬浮时间胶囊：

- 默认显示剩余时间、极细进度和运行/暂停状态；
- 点击或悬停后展开暂停、继续、结束 Session；
- 不把计时进度画在二级节点外环，避免与任务完成度混淆；
- 左上角的“返回 L2”和结束 Session 是两个独立动作。

## 7. 三级节点交互

### 单击

- 非当前三级：进入节点聚焦并设为当前项；
- 当前三级：保持聚焦，不触发隐藏的第二动作。

### 聚焦后显式入口

- `详情`：打开三级详情 Modal；
- `扩展画布`：进入窗口聚焦；
- 成果钩：形成/撤销本轮完成草稿。

不使用双击，不让点击节点本身同时承担完成或打开详情。

## 8. 成果完成草稿

- 成果钩位于当前三级节点边缘；
- 点击后写入 Session `completionDraft`，不立即修改正式 WorkItem；
- 可再次点击撤销；
- 视觉使用柔和内发光、闭合细光环和小型亮钩；
- Session 结束时统一提交正式完成命令；
- Note Checklist 勾选使用不同形状和视觉，不得混淆。

节点状态视觉：

| 状态 | 表达 |
|---|---|
| 普通 | 低饱和实色圆、稳定标题 |
| 当前 | 轻微前浮、细光环、对比提升 |
| 完成草稿 | 内发光、闭合光环、亮钩 |
| 阻塞 | 边缘断续环，不使用大面积红色 |
| 未触达 | 柔和暗沉，但保持文字可读 |

## 9. WorkItemNote 实时编辑

### 主路径

- 只有当前三级展开 Note；
- Note 通常显示一段正文；
- 节点聚焦态允许直接原位编辑；
- 停止输入约 800ms 后自动保存到正式 WorkItemNote；
- 失焦、返回 L2和缩回窗口前强制刷新；
- 在线显示极轻“已保存”，离线显示“待同步”。

### 快速追加

聚焦节点旁提供极简输入：

- 普通输入 → paragraph；
- `[]` 前缀 → checklist；
- `1.` 前缀 → ordered_list；
- `-` 前缀 → unordered_list。

### 展示策略

- 紧凑全景：其他三级仅显示 `Note · N`；
- 节点聚焦：展示当前三级最近一段 Note；
- 扩展态：最多投影 1–3 个 Note 摘要，其余显示“还有 N 条”；
- 完整结构编辑进入详情 Modal。

Session 随记继续独立，不自动沉淀到 WorkItemNote。

## 10. 三级详情 Modal

Modal 只承接低频完整信息：

- description；
- 完整结构化 WorkItemNote；
- WorkItem 正式状态；
- 阻塞与依赖信息；
- 本轮执行化身；
- 创建来源、最近历史及 Session 关联；
- 从本轮移除；
- 取消正式任务（二次确认）。

Modal 不重复：

- 成果完成草稿；
- 快速一段 Note 编辑；
- 暂停/结束 Session；
- 新建三级；
- Session 随记。

## 11. 执行化身

- 牛马、小猪、哈吉米、齐天大圣只在三级详情 Modal 中展示和编辑；
- 主画布不显示四选一按钮或化身图标；
- 化身仍属于 Session × 三级结果，不是负责人、WorkItem 状态或效率等级；
- 用户主动选择，系统可建议但不得预选。

## 12. 运行中新建三级

- 二级分支末端保留极简 `＋` 圆点；
- 点击后原位输入标题，回车创建正式三级；
- 创建后自动进入节点聚焦；
- 只要求标题，不要求估时、类型、Note 或 Cycle 角色；
- 来源记录为 `during_session`；
- 不自动改变新三级状态；
- 离线时入口保留但禁用，并解释“离线不能创建正式 WorkItem”。

## 13. 视觉系统

采用轻玻璃外壳＋实色节点：

- 窗口外壳：轻透明、低强度模糊、柔和阴影；
- WorkItem：近实色浅色圆节点；
- Note：略暖或略灰的实色方形节点；
- Modal：比外壳更实，优先保证阅读；
- 点亮依赖亮度、细光环和轻微位移，不依赖高饱和色；
- 保持浅色主题与深色文字；
- 不使用全玻璃拟态、霓虹、粒子或大面积渐变。

## 14. 动效

仅保留四类克制形变：

| 动效 | 建议时长 |
|---|---:|
| L2 → L3、紧凑 → 扩展 | 320–420ms |
| 节点聚焦与切换 | 180–240ms |
| Note 展开/收回 | 180–240ms |
| 完成草稿点亮/撤销 | 约 200ms |

只使用位移、缩放、透明度和柔和光环；支持 `prefers-reduced-motion`。

## 15. 原型必须覆盖的交互

1. 紧凑全景、节点聚焦、窗口聚焦三态；
2. 三级切换当前项；
3. 当前 Note 展开与防抖保存反馈；
4. 成果草稿点亮与撤销；
5. 原位新建三级；
6. “还有 N 项”候选加入；
7. 详情 Modal 与执行化身；
8. 从本轮移除和取消确认；
9. 顶部时间胶囊展开、暂停/继续；
10. 返回 L2 后计时继续，并可返回专注 L3；
11. 离线禁建三级与 Note 待同步；
12. 可移动小窗与主动放大/缩回。

## 16. 验收标准

- 用户第一眼把它理解为当前任务思维图，而不是任务管理工作台；
- 主画布中没有固定详情栏、成果列表栏、化身按钮组或底部 Dock；
- 用户能在两步内切换当前三级并编辑一段 Note；
- 用户能区分 Note 保存和成果完成草稿；
- 用户能随时返回 L2，且不会误以为 Session 已结束；
- 扩展画布不会丢失当前项、完成草稿或 Note；
- Modal 关闭后返回原聚焦位置；
- 节点数量增加时默认小窗不会自动侵占桌面；
- 视觉保持浅色、克制、可读，动画不干扰持续专注。

## 17. Non-goals

首版原型不验证：

- 最终 Tauri 多窗口性能；
- WebGL 或物理粒子效果；
- 化身自动识别；
- 三级分钟时间轴；
- 完整 Session 结束复盘；
- Cycle 工作台；
- 多人协作；
- 跨 Space 聚合；
- 完整富文本编辑器。
