# WorkItem 列表与详情工作面规格

> 状态：已批准，作为工程实施计划的产品基线  
> 版本：v1.0  
> 依赖：[共享领域契约](./WORKITEM_SHARED_CONTRACT.md)

## 1. 问题与目标

项目默认工作面需要同时支持快速捕捉、日常推进、柔性计划和关系理解，但不能把所有字段塞进表格。首屏必须优先回答：这是什么、现在怎样、准备何时投入、是否需要关注。

## 2. 非目标

- 不默认展示全部字段。
- 不在列表完成所有复杂关系编辑。
- 不把 L3 关系图嵌入每行。
- 不用大量红色表达柔性窗口偏差。
- 不建设独立于 WorkItem 的 Orbit 编辑表单。

## 3. 用户故事

- 作为高频记录者，我希望只输入标题就创建 WorkItem，以便不打断思路。
- 作为计划者，我希望列表直接看到窗口、投入和可行动信号，以便快速决定下一步。
- 作为复杂项目用户，我希望父子树与依赖信号分开表达，以便不混淆范围与阻塞。
- 作为键盘和移动端用户，我希望详情可恢复、可访问，以便不丢列表上下文。

## 4. 页面信息架构

```text
Project header
├─ Work items / Cycles / Modules / Views / Activity
├─ View toolbar
│  ├─ Filter
│  ├─ Group
│  ├─ Sort
│  └─ Display fields
└─ WorkItem workbench
   ├─ Quick create
   ├─ List projection
   └─ Detail drawer → Full page
```

## 5. 默认列表列

| 顺序 | 列 | 内容 | 行内编辑 |
|---|---|---|---|
| 1 | 层级/标题 | 展开、缩进、displayKey、标题、直接子项计数 | 标题 |
| 2 | Type | 图标 + 名称 | 是 |
| 3 | 状态 | Workspace 显示状态 | 是 |
| 4 | 计划 | 原始窗口短文案 + 硬截止徽标 | 是 |
| 5 | 投入 | 区间、约 5h、未知、需拆分 | 是 |
| 6 | 负责人 | 单人可隐藏，多人默认显示 | 是 |
| 7 | 信号 | 最多 2 个可行动信号 +N | 点击处理 |

Priority、Cycle、Module、Label、时间字段和递归进度为可选列。首版仅在当前 Project 会话中保留显示列、筛选、分组和排序；持久化 Saved View 属于 P1。当前折叠和选中仅为 session 状态。

## 6. 快速创建

- 默认只输入标题，Enter 创建。
- 自动填充 Project 默认 Type 和默认未开始状态。
- 计划、投入、信心、Cycle、Module、Label 默认空。
- 创建后显示短暂“补充计划”入口，不阻塞连续录入。
- 空白标题不创建；失败保留输入。
- 事实写入不等待 Orbit、索引或同步派生完成。
- 可展开高级创建覆盖 Type、状态和计划。

## 7. 计划列与信号列

### 7.1 计划列

- 文案：今天、未来三天、本周、近一周、下周、未来两周、本月、暂不安排、自定义范围。
- 只额外显示硬截止徽标；低信心、复盘、重排不在此重复。
- 自定义范围显示短日期；完整冻结日期通过聚焦/详情查看。
- 柔性窗口结束不用红色；硬截止已过可使用强警示。

### 7.2 信号列优先级

```text
硬约束 > 依赖阻塞/异常 > 需要重排 > 复盘到期 > 低信心
```

窗口未结束时可以出现复盘到期；一旦窗口结束，“需要重排”取代该任务的复盘到期，因此两者不会在同一 WorkItem 上竞争展示。

- 最多显示主信号和一个次信号；其余 `+N`。
- 每个信号提供名称、原因、建议动作。
- 不用综合风险分数替代原因。
- 同类排序使用客观时间/数量，仍相同时使用 `updatedAt`。
- 首版信号：硬截止已过、阻塞/依赖异常、复盘到期、需要重排、低信心。
- 容量超载和 planning uncertainty 属于 P1。

## 8. 父子、依赖与 Orbit

- 标题列只表达父子树，最多三层。
- 显示“直接完成数/有效直接子项数”；递归百分比在详情或可选进度列。
- 折叠父项保留子项信号摘要。
- 依赖不画进标题树；在信号列显示阻塞/外部依赖摘要。
- 点击依赖信号打开详情“关系”区。
- 点击关注信号打开“为什么关注”区，展示事实链。
- 从详情可进入以当前 WorkItem 为中心的 L3。
- 跨 Project 依赖在 L3 使用边界端口。

## 9. 详情形态

### 桌面

- 单击打开右侧宽抽屉，保留列表滚动、筛选和选中。
- 抽屉包含：标题/描述、Type/状态、计划、父子、关系、活动、为什么关注。
- 可进入全页详情处理长内容和复杂关系。

### 窄屏

- 直接进入全页详情。
- 返回恢复 View、筛选、分组、滚动、折叠和选中。
- 复杂 L3 使用专门全屏模式。

抽屉和全页共享信息架构；路由可刷新和复制。支持 Esc 关闭、上下项切换、焦点恢复；触控目标 ≥44px。

## 10. 详情字段布局

```text
Header: displayKey / title / status / actions
Summary: Type / assignee / labels
Planning: window / effort / confidence / review / hard deadline
Structure: parent / children / recursive progress
Relations: depends on / blocks / cross-project / related
Attention: signal reasons / suggested actions / open in L3
Content: description
Activity: event history
```

## 11. 批量操作

### P0

- 批量状态切换。
- 批量 Type、负责人、Label 修改。
- 批量归档/恢复。

### 约束

- 批量完成包含有活动子项的父项时逐项汇总异常，并要求统一或分别决策。
- 批量操作不得跳过状态、父子和依赖不变量。
- 部分失败时返回成功/失败清单，不回滚已明确成功的独立项，除非应用服务支持原子批次。

## 12. 空、加载与错误状态

- 空 Project：展示快速创建和模板建议，不展示虚假指标。
- 筛选无结果：展示清除筛选，不等同空 Project。
- 派生加载：事实行可操作，信号列显示“洞察更新中”。
- 写入失败：保留编辑值并可重试。
- 版本冲突：展示最新值与用户意图，允许重新应用，不静默覆盖。
- WorkItem 不可访问：详情显示不可用，不泄露标题和内容。

## 13. 键盘地图

| 操作 | 键盘 |
|---|---|
| 快速创建 | `C` 或聚焦创建行 |
| 打开详情 | Enter |
| 关闭抽屉 | Esc |
| 上/下一项 | `J` / `K`（输入框外） |
| 展开/折叠 | 左/右方向键 |
| 打开命令菜单 | `Ctrl/Cmd + K` |
| 保存编辑 | `Ctrl/Cmd + Enter` |

快捷键必须可发现、可关闭，且不得覆盖文本输入行为。

## 14. 事件与遥测

领域事件来自其他规格；工作台只发语义命令。产品遥测不记录标题和描述内容：

- `quick_create_started/succeeded/failed`
- `detail_opened`（drawer/fullpage）
- `signal_opened`（kind）
- `replan_action_selected`（action）
- `relation_section_opened`
- `l3_opened_from_workitem`
- `view_session_customized`

## 15. P0 / P1 / P2

### P0

- 默认列表、快速创建、行内 Type/状态/计划/投入。
- 七列默认布局与可选字段。
- 信号优先级和解释入口。
- 树形父子、折叠摘要、关系详情。
- 桌面抽屉/全页、窄屏全页。
- 空/加载/错误/冲突状态。
- 键盘与基本批量操作。

### P1

- 看板投影、用户保存 View。
- 递归进度列。
- 容量和 planning uncertainty 信号。
- 抽屉宽度偏好。

### P2

- 自定义快速创建字段。
- 高级表格公式和完整移动端应用。

## 16. 验收标准

1. **默认列**  
   Given首次进入 Project，Then显示七组核心列，不默认显示 Cycle/Module/Priority。
2. **快速创建**  
   Given输入标题回车，Then行立即出现，默认 Type/状态有效，计划为空。
3. **树形键盘操作**  
   Given父项有子项，When使用方向键，Then可展开/折叠且焦点不丢失。
4. **计划去重**  
   Given低信心且有硬截止，Then计划列显示窗口+硬截止，信号列显示低信心，不重复。
5. **重排取代复盘**  
   Given复盘已到且窗口后来结束，Then信号列显示需要重排，不同时展示过时复盘。
6. **信号原因**  
   When点击阻塞，Then展示具体依赖 WorkItem/Project、状态和建议动作，不只显示分数。
7. **折叠摘要**  
   Given折叠父项有两个阻塞子项，Then父行仍显示“2 个阻塞”摘要。
8. **详情恢复**  
   Given列表有筛选和滚动，When打开全页再返回，Then恢复上下文。
9. **响应式**  
   Given窄屏，When点击 WorkItem，Then进入全页而不是过窄抽屉。
10. **派生失败**  
    Given风险计算暂时失败，Then仍可编辑事实，信号列显示洞察更新中。
11. **版本冲突**  
    Given远端版本更新，When本地保存旧版本，Then不覆盖，展示冲突与重新应用入口。
12. **可访问性**  
    Given键盘和屏幕阅读器，Then可完成创建、打开详情、切状态、调整计划和查看信号原因。

## 17. 指标与验证假设

- 快速创建成功率 ≥98%，中位完成时间 <5 秒（待真实用户验证）。
- 详情返回上下文恢复率 100%。
- 信号点击后采取行动的比例用于评估可解释性，不作为用户绩效。
- 可访问性目标：核心流程 WCAG 2.1 AA。
- 研究任务：比较传统日期列与柔性窗口列对理解和焦虑的影响。

## 18. 开放问题

无阻塞产品问题。看板、完整 Saved View、容量信号和高级表格进入后续规格。
