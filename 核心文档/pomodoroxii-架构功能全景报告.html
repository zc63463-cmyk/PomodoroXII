PomodoroXII 是从 pomodoroxi 架构升级的全新全栈项目，核心定位是个人状态管理平台——以番茄钟为核心引擎，产出时间/专注/认知/行为四维元数据，供 Agent 和插件消费分析。

2. 设计理念与三条铁律

铁律 1：Routers commit

事务控制在请求边界。Router 负责调用 db.commit()，Service 只 flush。

铁律 2：Services 不导入 FastAPI

业务逻辑与 HTTP 解耦，可独立测试，不依赖 Request/Response。

铁律 3：Models 纯数据

ORM 模型只有字段和约束，无业务方法。SyncMixin 统一元数据。

Vibe Coding 时代的设计原则

原则核心思想PomodoroXII 对应

5. 同步引擎设计

核心机制

机制实现来源
Outbox 合并矩阵create+delete→消除, delete+create→update, update+update→取最新pomodoroxi 战斗经验
LWW + content_hash双游标增量：updated_at 检测元数据变化, content_hash 检测正文变化本次升级
Tombstone 防复活TTL 90d + 墓碑检查 + created_at 比较 + TTL 过期保护pomodoroxi
零点时间戳检测识别 00:00:00.000 为陈旧数据，跳过 LWW 冲突pomodoroxi
SAVEPOINT 隔离每事件独立事务，单失败不滚全批pomodoroxi
Saga 跨库一致性先 DB flush → 再 FS write → 统一 commit → 失败补偿本次新建
5 道安全检查墓碑/零点时间戳/字段剥离/循环引用/Tombstone 创建提取为 sync_safety.py
sync 双路径note/folder 走 NoteFileService，其他 12 种走 ORM本次设计

8. 可靠性保障

同步可靠性

  • Outbox 合并矩阵（create+delete 消除）
  • Tombstone 防复活（TTL 90d + 5 道检查）
  • SAVEPOINT 隔离（单事件失败不滚全批）
  • 死信处理（不可重试错误自动移出 outbox）
  • Promise mutex 串行化（sync/fullSync 不并发）

数据可靠性

  • 双库自动备份（每日 3:00）
  • 定时 JSONL 快照（每日 3:30）
  • 一致性检查与修复（每小时）
  • .md 原子写入（临时文件 + os.replace）
  • 三层锁（RLock → FileLock → SQLite 事务）

已知风险与缓解

风险严重度缓解措施
跨库一致性（DB + .md）高Saga + 补偿操作 + 定时一致性检查
sync_pull 读 .md 性能中read_notes_batch 批量读取 + content_hash 跳过
LWW 依赖客户端时钟中content_hash 缓解 + HLC（Phase 3 可选）
万条笔记以上规模中舒适区万条以内，十万条需文件分片
