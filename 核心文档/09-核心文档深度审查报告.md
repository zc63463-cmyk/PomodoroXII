### 5.2 新增 React 迁移耦合点（本次重构新增）

React 19 + Next.js 15 + Zustand + TanStack Query 的引入，在前端层引入了 3 类新的耦合风险。这些风险在后端同步引擎层面不存在，但会直接影响离线一致性、渲染性能与组件边界正确性。

| 耦合点 | 涉及功能 | 耦合类型 | 风险 | 建议与约束 |
|--------|---------|---------|------|------|
| **Zustand store 间依赖（selector 嵌套）** | 跨域 store（如 I1↔D5 日程-速记、J2↔F4 打卡-活动日历、B1↔A1 任务-会话） | 状态依赖 | 中 | 跨 store 调用通过 `get()` 显式注入，不在 selector 中直接订阅其他 store；使用 `shallow` 比较避免循环依赖与多余 re-render；严禁 selector 嵌套导致的渲染级联。12 文档 AGENTS.md 模板已写入"Zustand store 必须用 selector 访问状态，不要解构整个 store"约束 |
| **TanStack Query cache 与 Dexie 同步策略** | F/G（统计/同步）↔ Dexie 离线存储 | 缓存一致性 | 中 | `useQuery` 优先读 Dexie 离线缓存，`useMutation` 乐观更新后 `invalidateQueries`；`gcTime`（30min）与 Dexie 保留期对齐；Outbox 队列（`_dirty` 标记）作为 cache miss 兜底。02 文档第 3.4 节已给出 `useTasks` 的 Dexie 优先 + 乐观更新范式，03 文档第 3.2 节已将此耦合点纳入分析表 |
| **Server/Client Component 边界混淆风险** | 全局（`app/**/page.tsx` 默认 Server Component vs `'use client'` 组件） | 渲染边界 | 高 | Server Component 为默认，需交互/hooks 的组件在文件顶部声明 `'use client'`；Client Component 不可直接导入 Server Component 内部模块，只能通过 `children` prop 传递；Server Component 传递给 Client Component 的 props 必须可序列化。12 文档 AGENTS.md 的 `Never Do` 已写入此约束，01 文档 M5b 验证项含 Server/Client 边界划分检查 |

#### 5.2.1 Zustand selector 嵌套风险详解

```ts
// ❌ 错误：selector 内直接订阅其他 store，导致渲染级联
const habits = useHabitStore(s => s.habits)
const checkIns = useCheckInStore(s => s.checkIns) // 在另一个组件的 selector 中隐式依赖

// ✅ 正确：跨 store 数据通过 action 显式注入，selector 只读自身 store
const useActivityCalendar = () => {
  const sessions = useSessionStore(s => s.sessions, shallow)
  const checkIns = useCheckInStore(s => s.checkIns, shallow)
  // 在组件层组合，而非在 selector 内嵌套跨 store 订阅
  return useMemo(() => mergeCalendarData(sessions, checkIns), [sessions, checkIns])
}
```

**风险场景**：F4 活动日历（A1 会话 + J2 打卡）天然跨 store，若在 selector 内嵌套订阅，任一 store 更新都会触发全量 re-render。

#### 5.2.2 TanStack Query ↔ Dexie 同步策略详解

| 场景 | 在线 | 离线 | 处理方式 |
|------|------|------|---------|
| 读（`useQuery`） | Dexie 命中即返回，后台 `refetch` 拉取服务端增量 | Dexie 命中即返回，无 refetch | cache miss 时读空，由 Outbox 兜底 |
| 写（`useMutation`） | `onMutate` 先写 Dexie + `_dirty=true` + 更新 Query cache | 同左，但 push 入 Outbox 队列等待联网 | 联网后 `useSyncOutbox` 批量推送 |
| 回滚（`onError`） | 恢复 Dexie 旧值 + 恢复 Query cache | 同左 | 保证离线下乐观更新可回滚 |
| GC（`gcTime` 30min） | 过期缓存清理，Dexie 仍保留 | 同左 | `gcTime` 与 Dexie 保留期对齐，避免缓存清理后离线无数据 |

#### 5.2.3 Server/Client Component 边界详解

| 组件类型 | 可用能力 | 限制 | 在 PomodoroXII 的典型用法 |
|---------|---------|------|-------------------------|
| Server Component（默认） | `async` 数据获取、直接访问后端资源 | 不能用 `useState`/`useEffect`/handlers，不能导入 Zustand/TanStack Query | `app/tasks/page.tsx` 列表页（服务端拉取初始数据） |
| Client Component（`'use client'`） | 全部 React hooks、Zustand、TanStack Query、事件处理 | 不能 `async export default`，不能直接导入 Server Component 内部模块 | `components/task/TaskEditor.tsx`（交互编辑器） |

**关键约束**：Server → Client 只能传**可序列化 props**（不可传函数/类实例/React 元素）；Client 需要服务端数据时，由 Server Component 获取后通过 props 下发，或由 Client Component 自身用 TanStack Query 拉取。
