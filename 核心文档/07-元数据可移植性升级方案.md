## 二、行业参考项目

### 2.1 MCP (Model Context Protocol) — Agent 数据访问的标准协议

**核心价值**：MCP 是 2025-2026 年 AI Agent 与数据源交互的事实标准协议。Claude Desktop、Cursor、VSCode 等 AI 客户端原生支持 MCP。

**MCP 三大概念**：

| 概念 | 用途 | Pomodoroxi 对应 |
|------|------|----------------|
| **Tools** | Agent 可调用的函数 | `query_tasks(status="done")`, `search_notes("专注力")` |
| **Resources** | Agent 可读取的只读数据 | `px://notes/n_abc123`, `px://stats/weekly/2026-06-30` |
| **Prompts** | 预定义交互模板 | "分析本周专注力趋势", "生成每日复盘" |

**可借鉴设计**：

```typescript
// MCP Server for Pomodoroxi

  status: z.string().optional(),
  priority: z.array(z.string()).optional(),

}, async (params) => {

server.resource("note", new ResourceTemplate("px://notes/{id}"),

  });

server.resource("weekly_stats", new ResourceTemplate("px://stats/weekly/{date}"),
  async (uri, { date }) => {

  });
```

**学习项目**：https://github.com/modelcontextprotocol/servers

### 2.2 Joplin REST API — 笔记应用数据访问的标杆

**核心价值**：Joplin 通过本地 REST API 让任何外部程序都能结构化访问笔记数据。

**设计要点**：

| 特性 | Joplin 实现 | Pomodoroxi 可借鉴度 |
|------|------------|-------------------|
| 资源路由 | `/notes`, `/folders`, `/tags`, `/search` | ⭐⭐⭐⭐⭐ |
| Token 认证 | `?token=xxx` 或 `Authorization: Bearer xxx` | ⭐⭐⭐⭐ 已有 JWT |
| 标准响应 | `{status, data, meta:{pagination}}` | ⭐⭐⭐⭐ |
| 全文搜索 | `GET /search?query=xxx&type=note` | ⭐⭐⭐⭐⭐ FTS5 已有 |
| 事件流 | `GET /events?cursor=xxx` 追踪变更 | ⭐⭐⭐⭐⭐ |
| 分页 | `?limit=50&page=2&order_by=created_time` | ⭐⭐⭐⭐ |

**最值得借鉴的是事件流 API**：

```
GET /events?cursor=last_event_id
→ 返回自 cursor 以来的所有变更事件（create/update/delete）
```

这正好补上了 Pomodoroxi 缺的"实时变更推送"——Agent 可以轮询 `/events` 获取增量变更，不需要全量拉取。

### 2.3 Tolaria — Files-first + Git-first 的可移植性验证

**核心价值**：Tolaria（12K+ stars）证明了"纯 .md 文件 + git"的模式在万级笔记下可行，且对 AI 工具天然友好。

**关键设计**：

- `AGENTS.md` — vault 根目录的 AI 导航文件，告诉 Claude Code / Cursor 如何读取 vault
- YAML frontmatter — 软约束的类型系统，不强制但可解析
- git history — 天然版本管理，无需自建

**可借鉴**：在 `data/notes/` 根目录放一个 `AGENTS.md`，引导 AI 工具理解数据结构。

from apscheduler.schedulers.asyncio import AsyncIOScheduler

class SnapshotService:
    """定时将元数据快照到文件系统"""

    EXPORT_DIR = Path("data/export")

    async def daily_snapshot(self):
        """每天凌晨 3:00 执行全量快照"""
        self.EXPORT_DIR.mkdir(parents=True, exist_ok=True)
        date_str = datetime.utcnow().strftime("%Y-%m-%d")

        # 1. 各实体 JSONL 快照
        for entity in ("tasks", "sessions", "notes", "reflections", "habits", "folders"):
            await self._export_entity_jsonl(entity, date_str)

        # 2. 关系图快照
        await self._export_graph(date_str)

        # 3. 分析提示快照
        await self._export_analysis_hints(date_str)

        # 4. 清理 30 天前的快照
        self._cleanup_old_snapshots(30)

    async def _export_entity_jsonl(self, entity: str, date_str: str):
        """导出单实体为 JSONL"""
        path = self.EXPORT_DIR / f"{entity}.jsonl"
        # 流式写入，每行一个 JSON
        ...

    async def _export_graph(self, date_str: str):
        """导出关系图"""
        path = self.EXPORT_DIR / "relations.jsonl"
        # nodes + edges
        ...

    def _cleanup_old_snapshots(self, keep_days: int):
        """清理旧快照"""
        cutoff = datetime.utcnow().timestamp() - (keep_days * 86400)
        for f in self.EXPORT_DIR.glob("*.jsonl"):
            if f.stat().st_mtime < cutoff:
                f.unlink()

# main.py lifespan 中注册
scheduler = AsyncIOScheduler()
scheduler.add_job(
    SnapshotService().daily_snapshot,
    'cron', hour=3, minute=0
)
scheduler.start()

## 六、总结

你现有的 `export-v2.ts` 已经设计了 `_graph`(关系图) + `_analysis_hints`(分析提示) + NDJSON 流式格式，这三个要素对 Agent 极其友好。这三个模块为框架无关 TypeScript，前端迁移到 React 19 后原样复用，无需重写。缺的不是设计，而是：

1. **后端没有对应实现** — export-v2.ts 跑在 React 19 前端（框架无关，Agent 无法直接访问），需在后端用 Python 重新实现同一格式契约
2. **没有自动化** — 需要用户手动点导出
3. **.md 文件没有 frontmatter** — Agent 读文件只看到正文

升级方案的四层访问体系（全部后端实现，与前端框架无关）：

- **MCP Server**：Claude/Cursor 原生接入，Tools + Resources
- **REST Export API**：程序化接入，复用 export-v2.ts 格式契约
- **.md frontmatter**：文件自描述，Agent 直接读文件
- **定时快照**：data/export/*.jsonl 每日自动更新

四层互补：实时场景用 MCP/API，离线场景读文件/快照。Agent 可以根据场景选择最合适的路径。前端框架从 Vue 3 替换为 React 19 不影响本方案任何后端设计——`export-v2.ts` / `export-arrow.ts` / `export-memo.ts` 作为框架无关代码原样复用，仅复盘导出 UI 组件需重写为 React。
