## 需求 1：复盘系统文件化归档 + 结构化元数据

### 1.1 需求描述

复盘（Reflection）数据目前全部存储在 DB 的 reflections 表中。随着时间推移，历史复盘成为冷数据但永远不会被删除，导致 DB 表持续膨胀。用户希望：

1. 复盘采用 **.md 文件 + YAML frontmatter** 存储，与笔记系统保持一致的设计语言
2. frontmatter 包含结构化元数据，包括当日 session 快照数据
3. 超过一定期限的复盘自动归档为只读，DB 中只保留轻量索引
4. 归档复盘支持"恢复编辑"操作

### 1.2 当前状态

#### 1.3.3 热冷分层策略

| 层级 | 存储位置 | 数据 | 可编辑 | sync |
|------|---------|------|--------|------|
| **热层** | pomodoroxi.db reflections 表 | 最近 N 天（默认 30 天，可配置） | ✅ 可编辑 | ✅ 走 sync push/pull |
| **冷层** | data/reflections/YYYY-MM/YYYY-MM-DD.md | 超过 N 天的归档复盘 | ❌ 只读（可恢复编辑） | ❌ 不走 sync（文件即持久化） |

**DB 中归档记录**：冷层的复盘在 DB 中保留一条轻量索引记录：

```python
class Reflection(Base, SyncMixin):
    # ... 原有字段 ...
    archived: bool = False                    # 新增：是否已归档
    archived_path: str | None = None          # 新增：.md 文件相对路径
    session_snapshot: str | None = None       # 新增：当日 session 快照 JSON
```

归档后，`content`、`sections`、`tags` 等大字段可以清空（或保留为摘要），只有 `id`、`date`、`mood`、`archived_path` 保留在 DB 中供列表查询。

#### 1.3.4 session 快照机制

**核心原则**：复盘写入时冻结当日 session 统计，后续 session 变更不影响历史复盘的真实性。

```python
# app/services/reflection_service.py
async def create_or_update_reflection(self, reflection_id, date, ...):
    # 1. 生成当日 session 快照
    snapshot = await self._build_session_snapshot(date)
    # 2. 写入 .md 文件（frontmatter 含 snapshot）
    md_content = build_reflection_md(meta, content, snapshot)
    await self.fs.write_reflection(date, md_content)
    # 3. 写入 DB（含 snapshot JSON）
    reflection = Reflection(..., session_snapshot=json.dumps(snapshot))
    ...

async def _build_session_snapshot(self, date: str) -> dict:
    """从 DB 查询当日所有 session，生成统计快照"""
    sessions = await self.db.execute(
        select(PomodoroSession)
        .where(PomodoroSession.started_at.like(f"{date}%"))
        .order_by(PomodoroSession.started_at)
    )
    sessions = sessions.scalars().all()
    completed = [s for s in sessions if s.completed and s.type == "work"]
    return {
        "date": date,
        "total_sessions": len(sessions),
        "completed_sessions": len(completed),
        "total_focus_minutes": sum(s.duration for s in completed) // 60,
        "avg_attention_score": sum(s.attention_score or 0 for s in completed) / max(len(completed), 1),
        "flow_states": sum(1 for s in completed if s.flow_state),
        "tasks_worked_on": [
            {"id": t.id, "title": t.title, "pomodoros": count}
            for t, count in task_pomodoro_counts(sessions)
        ],
        "top_task_id": most_focused_task_id(sessions),
        "mood_distribution": count_by_mood(sessions),
    }
```

#### 1.3.5 归档自动化

```python
# app/services/reflection_service.py
ARCHIVE_AFTER_DAYS = 30  # 可配置

async def auto_archive_old_reflections(self):
    """定时任务：归档超过 N 天的复盘"""
    cutoff_date = (utc_now() - timedelta(days=ARCHIVE_AFTER_DAYS)).strftime("%Y-%m-%d")
    old_reflections = await self.db.execute(
        select(Reflection)
        .where(Reflection.date < cutoff_date, Reflection.archived == False)
    )
    for r in old_reflections.scalars():
        # 1. 写 .md 文件（如果尚未写入）
        md_content = build_reflection_md(r)
        rel_path = f"{r.date[:7]}/{r.date}.md"
        await self.fs.write_reflection(rel_path, md_content)
        # 2. 更新 DB 记录
        r.archived = True
        r.archived_path = rel_path
        # 可选：清空大字段减少 DB 体积
        # r.content = ""
        # r.sections = "[]"
    await self.db.flush()
```

#### 1.3.6 只读 + 恢复编辑

**前端：归档复盘展示为只读 + 恢复编辑**

状态管理从 Pinia 迁移为 Zustand 5，组件从 Vue SFC 迁移为 React 19 JSX。核心变化：

| Vue 3 + Pinia（pomodoroxi） | React 19 + Zustand（PomodoroXII） |
|------------------------------|-------------------------------------|
| `import { useReflectionStore } from '@/stores/reflection'` | `import { useReflectionStore } from '@/stores/reflection-store'` |
| `const store = useReflectionStore()` | `const reflections = useReflectionStore((s) => s.reflections)` |
| `storeToRefs(store)` 解构响应式 | selector 函数按需订阅，避免整体 re-render |
| `:reflection="r"` `:readonly="r.archived"` | `reflection={r}` `readonly={r.archived}` |
| `@restore="restoreReflection(r.id)"` | `onRestore={() => restoreReflection(r.id)}` |

Zustand store 定义：

```tsx
// frontend/src/stores/reflection-store.ts
import { create } from 'zustand'
import { devtools, persist } from 'zustand/middleware'
import { api } from '@/lib/api'
import type { Reflection } from '@/types'

interface ReflectionState {
  reflections: Reflection[]
  restoreReflection: (id: string) => Promise<void>
}

export const useReflectionStore = create<ReflectionState>()(
  devtools(
    persist(
      (set) => ({
        reflections: [],

        restoreReflection: async (id) => {
          // 调用后端 API 恢复编辑
          await api.post(`/reflections/${id}/restore`)
          // 更新本地状态：标记为可编辑
          set((state) => ({
            reflections: state.reflections.map((r) =>
              r.id === id
                ? { ...r, archived: false, archived_path: null }
                : r
            ),
          }))
        },
      }),
      { name: 'reflection-store' }
    )
  )
)
```

组件使用 — selector 按需订阅，归档复盘渲染为只读卡片：

```tsx
// frontend/src/components/reflection/reflection-list.tsx
'use client'
import { useReflectionStore } from '@/stores/reflection-store'
import { ReflectionCard } from '@/components/reflection/reflection-card'

export function ReflectionList() {
  // Zustand selector — 只订阅 reflections 数组，避免整体 re-render
  const reflections = useReflectionStore((s) => s.reflections)
  const restoreReflection = useReflectionStore((s) => s.restoreReflection)

  return (
    <div className="space-y-4">
      {reflections.map((r) => (
        <ReflectionCard
          key={r.id}
          reflection={r}
          readonly={r.archived}
          onRestore={() => restoreReflection(r.id)}
        />
      ))}
    </div>
  )
}
```

**后端：恢复编辑**

```python
# app/services/reflection_service.py
async def restore_reflection(self, reflection_id: str):
    """从 .md 文件恢复复盘到可编辑状态"""
    r = await self.get(Reflection, reflection_id)
    if not r.archived:
        raise ConflictError("Reflection is not archived")
    # 从 .md 文件读取完整内容
    md_content = await self.fs.read_reflection(r.archived_path)
    meta, body = parse_frontmatter(md_content)
    # 恢复 DB 字段
    r.content = body
    r.sections = json.dumps(meta.get("sections", []))
    r.archived = False
    r.archived_path = None
    await self.db.flush()
```

### 1.4 实施要点

| 要点 | 说明 |
|------|------|
| **file_system 扩展** | 新增 `reflection_ops.py` Mixin，提供 write_reflection / read_reflection / list_reflections 方法 |
| **frontmatter 复用** | 复用笔记的 `frontmatter.py` 模块，YAML 读写逻辑一致 |
| **FTS5 搜索** | 复盘 .md 文件也可以加入 FTS5 索引，实现"搜索历史复盘中的关键词" |
| **sync 适配** | 热层复盘走 sync push/pull（与其他实体一致）；冷层复盘不走 sync（文件即持久化，跨设备通过 file_system 同步或手动导出） |
| **session 快照时机** | 复盘创建/更新时生成快照，不在 session 变更时更新（保证历史真实性） |
| **Agent 可读** | Agent 读 `data/reflections/2026-07/2026-07-01.md` 就能同时拿到结构化元数据和反思正文 |

### 1.5 数据迁移

从 pomodoroxi 迁移到 PomodoroXII 时，需要一次性迁移脚本：

```python
async def migrate_reflections_to_files(db, fs):
    """将现有 DB 中的复盘写入 .md 文件"""
    result = await db.execute(select(Reflection))
    for r in result.scalars():
        snapshot = await build_session_snapshot(db, r.date)
        md_content = build_reflection_md(r, snapshot)
        rel_path = f"{r.date[:7]}/{r.date}.md"
        await fs.write_reflection(rel_path, md_content)
        r.session_snapshot = json.dumps(snapshot)
        # 30 天前的标记为已归档
        if r.date < (utc_now() - timedelta(days=30)).strftime("%Y-%m-%d"):
            r.archived = True
            r.archived_path = rel_path
    await db.flush()
```
