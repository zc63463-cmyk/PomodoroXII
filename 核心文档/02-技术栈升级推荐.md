## 一、升级总览

### 核心原则

后端保留 FastAPI 三层架构升级路线，前端从 Vue 3 生态全面替换为 React 19 + Next.js 15 生态。pomodoroxi 的 sync 引擎战斗经验在后端直接复用，前端约 30% 框架无关代码（services/types/utils/workers）也可直接迁移，70% 组件层重写为 React。采用微前端渐进迁移策略（方案 B），非大爆炸重写。

## 二、后端升级详情

### 2.1 包管理：pip → uv

**为什么升级**：uv 比 pip 快 10-100 倍，有 lockfile 保证可重现构建，pyproject.toml 是 Python 打包的现代标准。

### 2.1.1 认证依赖修正：python-jose + passlib → PyJWT + bcrypt

**问题**：pomodoroxi 使用 `python-jose 3.4.0`（最后一次发布 2022 年，项目不活跃，有 CVE-2024-33664）+ `passlib[bcrypt] 1.7.4`（代码中实际未使用 passlib，直接调用 bcrypt，且 passlib 1.7.4 与 bcrypt 4.x 不兼容会产生 deprecation warning）。

**修正**：

```diff
- python-jose[cryptography]==3.4.0   # 过时，有安全漏洞
- passlib[bcrypt]==1.7.4              # 死依赖，代码未使用
+ pyjwt>=2.10                         # 活跃维护，API 几乎一致
+ bcrypt>=4.2                         # 直接调用，无需 passlib 封装
```

```python
# 修正后的 security.py

from jwt.exceptions import InvalidTokenError  # 替代 from jose import JWTError

def create_access_token(data: dict, expires_delta=None) -> str:
    to_encode = data.copy()
    expire = utc_now() + (expires_delta or timedelta(minutes=settings.jwt_expire_minutes))
    to_encode.update({"exp": expire})
    return jwt.encode(to_encode, settings.secret_key, algorithm=settings.jwt_algorithm)

def decode_access_token(token: str) -> dict | None:
    try:
        return jwt.decode(token, settings.secret_key, algorithms=[settings.jwt_algorithm])
    except InvalidTokenError:
        return None
```

**关键原则**：
- **Routers 提交（commit），Services 刷新（flush）** — Service 不知道自己在请求中是否是唯一操作
- **Services 不导入 FastAPI** — 业务逻辑与 HTTP 解耦，可独立测试
- **Models 纯数据** — ORM 模型只有字段和约束，无业务方法

### 2.4 Service 层模式

```python
# app/services/base.py
class BaseService:
    def __init__(self, db: AsyncSession):
        self.db = db

    async def get(self, model_cls, entity_id: str):
        result = await self.db.execute(
            select(model_cls).where(model_cls.id == entity_id)
        )
        obj = result.scalar_one_or_none()
        if not obj:
            raise NotFoundError(model_cls.__name__, entity_id)
        return obj

    async def create(self, model_cls, data: dict):
        obj = model_cls(**data)
        self.db.add(obj)
        await self.db.flush()  # flush 不 commit
        await self.db.refresh(obj)
        return obj

    async def update(self, obj, data: dict):
        for key, value in data.items():
            setattr(obj, key, value)
        await self.db.flush()
        return obj

# app/services/task_service.py
class TaskService(BaseService):
    async def create_task(self, data: dict) -> Task:
        return await self.create(Task, data)

    async def update_task(self, task_id: str, data: dict) -> Task:
        task = await self.get(Task, task_id)
        return await self.update(task, data)

    async def delete_task(self, task_id: str):
        task = await self.get(Task, task_id)
        await CascadeService(self.db).delete_task_cascade(task_id)
        await self.db.flush()

# app/routes/v1/tasks.py — Router 只做校验 + 调 Service + commit
@router.post("", response_model=TaskResponse, status_code=201)
async def create_task(payload: TaskCreate, db: DBSession) -> TaskResponse:
    svc = TaskService(db)
    task = await svc.create_task(payload.model_dump())
    await db.commit()  # Router 负责 commit
    return TaskResponse.model_validate(task)
```

### 2.6 结构化日志 + request_id

```python
# app/logging.py
import json, logging, time, uuid
from contextvars import ContextVar

request_id_var: ContextVar[str] = ContextVar("request_id")

class JsonFormatter(logging.Formatter):
    def format(self, record):
        payload = {
            "ts": int(time.time() * 1000),
            "level": record.levelname,
            "logger": record.name,
            "msg": record.getMessage(),
        }
        if request_id_var.get(None):
            payload["request_id"] = request_id_var.get()
        if record.exc_info:
            payload["exc"] = self.formatException(record.exc_info)
        return json.dumps(payload)

# request_id 中间件
@app.middleware("http")
async def request_id_middleware(request: Request, call_next):
    rid = request.headers.get("x-request-id") or uuid.uuid4().hex[:8]
    token = request_id_var.set(rid)
    try:
        response = await call_next(request)
        response.headers["x-request-id"] = rid
        return response
    finally:
        request_id_var.reset(token)
```

**为什么引入**：React 生态中服务端状态管理的标杆。配合 Dexie.js 离线同步，提供缓存、乐观更新、后台刷新、请求去重。v6 支持信号原生缓存，同时兼容 React 和 Vue。

```tsx

export const queryClient = new QueryClient({

// 在组件中使用
'use client'
import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query'

export function useTasks() {
  return useQuery({
    queryKey: ['tasks'],
    queryFn: async () => {
      // 先读 Dexie 离线缓存
      const local = await db.tasks.toArray()
      if (local.length > 0) return local
      // 再从服务端拉取
      const res = await api.get('/tasks')
      return res.data
    },
  })
}

export function useCreateTask() {
  const qc = useQueryClient()
  return useMutation({
    mutationFn: (data: Partial<Task>) => api.post('/tasks', data),
    onMutate: async (data) => {
      // 乐观更新：先写 Dexie + 更新 UI
      await db.tasks.add({ ...data, id: nanoid(), _dirty: true })
      await qc.cancelQueries({ queryKey: ['tasks'] })
      const previous = qc.getQueryData(['tasks'])
      qc.setQueryData(['tasks'], (old) => [...old, data])
      return { previous }
    },
    onError: (_err, _data, context) => {
      // 回滚
      qc.setQueryData(['tasks'], context?.previous)
    },
    onSettled: () => {
      qc.invalidateQueries({ queryKey: ['tasks'] })
    },
  })
}
```

### 3.5 Zustand 5 — 客户端状态管理

**为什么选择 Zustand 而非 Redux**：Zustand 心智模型最接近 Pinia（store 是一个 hook，直接 get/set），迁移成本最低。无 Provider 包裹，无 action/reducer 样板代码，bundle 体积仅 ~1KB。Redux Toolkit 虽然是官方推荐，但心智迁移成本高（Pinia 的 `state`/`getters`/`actions` → Redux 的 `createSlice`/`configureStore`/`dispatch`）。

**Pinia → Zustand 迁移映射**：

| Pinia 概念 | Zustand 等价 | 说明 |
|-----------|-------------|------|
| `defineStore('name', () => {...})` | `create()(set, get) => ({...})` | store 创建 |
| `const count = ref(0)` | `count: 0` | state |
| `const double = computed(() => count.value * 2)` | `double: () => get().count * 2` | getter |
| `function increment() { count.value++ }` | `increment: () => set({ count: get().count + 1 })` | action |
| `const store = useStore()` | `const count = useStore(s => s.count)` | selector |
| `storeToRefs(store)` | `useStore()` + selector | 解构 |

```tsx
// frontend/src/stores/task-store.ts
import { create } from 'zustand'
import { devtools, persist } from 'zustand/middleware'

interface TaskState {
  tasks: Task[]
  selectedId: string | null
  filter: TaskFilter

  // actions
  addTask: (task: Task) => void
  updateTask: (id: string, patch: Partial<Task>) => void
  deleteTask: (id: string) => void
  setSelected: (id: string | null) => void
  setFilter: (filter: TaskFilter) => void
}

export const useTaskStore = create<TaskState>()(
  devtools(
    persist(
      (set, get) => ({
        tasks: [],
        selectedId: null,
        filter: 'all',

        addTask: (task) => set((state) => ({ tasks: [...state.tasks, task] })),
        updateTask: (id, patch) => set((state) => ({
          tasks: state.tasks.map((t) => t.id === id ? { ...t, ...patch } : t),
        })),
        deleteTask: (id) => set((state) => ({
          tasks: state.tasks.filter((t) => t.id !== id),
          selectedId: state.selectedId === id ? null : state.selectedId,
        })),
        setSelected: (id) => set({ selectedId: id }),
        setFilter: (filter) => set({ filter }),
      }),
      { name: 'task-store' }
    )
  )
)

// 在组件中使用 — selector 避免不必要的 re-render
'use client'
import { useTaskStore } from '@/stores/task-store'

export function TaskList() {
  const filter = useTaskStore((s) => s.filter)
  const tasks = useTaskStore((s) => {
    switch (s.filter) {
      case 'active': return s.tasks.filter((t) => !t.completed)
      case 'completed': return s.tasks.filter((t) => t.completed)
      default: return s.tasks
    }
  })
  const setSelected = useTaskStore((s) => s.setSelected)

  return (
    <ul>
      {tasks.map((task) => (
        <li key={task.id} onClick={() => setSelected(task.id)}>
          {task.title}
        </li>
      ))}
    </ul>
  )
}
```

### 3.7 React Hook Form + Zod — 表单验证

**为什么引入**：Vue 的 v-model 双向绑定在 React 中没有等价物。React Hook Form 是 React 表单管理的标杆，非受控注册模式性能最优。Zod 提供运行时类型验证，与 TypeScript 类型系统联动。

```tsx
// frontend/src/lib/validations/task.ts
import { z } from 'zod'

export const taskSchema = z.object({
  title: z.string().min(1, '标题不能为空').max(200, '标题过长'),
  description: z.string().max(2000).optional(),
  priority: z.enum(['low', 'medium', 'high']),
  estimatedPomodoros: z.number().int().min(1).max(20),
  folderId: z.string().nullable(),
})

export type TaskFormData = z.infer<typeof taskSchema>

// frontend/src/components/task-form.tsx
'use client'
import { useForm } from 'react-hook-form'
import { zodResolver } from '@hookform/resolvers/zod'
import { taskSchema, TaskFormData } from '@/lib/validations/task'
import { Form, FormControl, FormField, FormItem, FormLabel, FormMessage } from '@/components/ui/form'
import { Input } from '@/components/ui/input'
import { Button } from '@/components/ui/button'

export function TaskForm({ onSubmit }: { onSubmit: (data: TaskFormData) => void }) {
  const form = useForm<TaskFormData>({
    resolver: zodResolver(taskSchema),
    defaultValues: {
      title: '',
      description: '',
      priority: 'medium',
      estimatedPomodoros: 1,
      folderId: null,
    },
  })

  return (
    <Form {...form}>
      <form onSubmit={form.handleSubmit(onSubmit)} className="space-y-4">
        <FormField
          control={form.control}
          name="title"
          render={({ field }) => (
            <FormItem>
              <FormLabel>标题</FormLabel>
              <FormControl>
                <Input {...field} />
              </FormControl>
              <FormMessage />
            </FormItem>
          )}
        />
        <Button type="submit">创建任务</Button>
      </form>
    </Form>
  )
}
```

## 六、不升级什么（以及为什么）

| 技术 | 不升级原因 |
|------|-----------|
| ~~SQLModel~~ | SQLModel 虽然简化了 Pydantic + SQLAlchemy 集成，但它锁定了 SQLAlchemy 版本、隐藏了太多细节、社区有性能争议。SQLAlchemy 2.0 Mapped[T] 已经足够好 |
| ~~Svelte/Solid~~ | React 19 生态最大，AI 编程工具（Cursor/Claude/Copilot）支持最好。Svelte 5 虽然性能优秀但生态和 AI 支持不足，团队学习成本高 |
| ~~Redux/Redux Toolkit~~ | Zustand 心智模型最接近 Pinia（store 是一个 hook，直接 get/set），迁移成本最低。Redux 的 action/reducer/dispatch 模式对 Pinia 用户来说心智迁移成本高 |
| ~~PostgreSQL~~ | SQLite WAL + busy_timeout 在单用户场景下完全够用，PostgreSQL 增加运维复杂度 |
| ~~CRDT (Y.js/Automerge)~~ | 单用户离线同步不需要 CRDT，LWW + content_hash 已足够 |
| ~~Redis~~ | 单用户不需要 Redis 做限流/缓存，内存限流够用 |
| ~~GraphQL~~ | REST + sync 端点已覆盖所有需求，GraphQL 增加复杂度 |
| ~~Vite (保留)~~ | Next.js 15 内置 Turbopack 构建已足够。如果项目不需要 SSR，也可用 Vite + @vitejs/plugin-react，但会失去 Server Components 优势 |
| ~~Dexie.js (保持)~~ | 离线存储引擎框架无关，v15 schema 已稳定，无需更换 |
| ~~Tailwind CSS (保持)~~ | 原子化 CSS 跨框架通用，class 可直接复制 |
