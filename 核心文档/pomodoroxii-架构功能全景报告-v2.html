⚙ 设计理念

三条架构铁律 + Vibe Coding 五原则，从第一天就确立的代码组织哲学

⚙ Routers commit, Services flush

事务控制权在请求边界（Router），Service 只 flush 不 commit。SAVEPOINT 隔离不会被击穿。

📝 Services 不导入 FastAPI

业务逻辑与 HTTP 解耦。Service 可以独立单元测试，也可以被 CLI / 定时任务 / MCP Server 复用。

💾 Models 纯数据

ORM 模型只有字段和约束，无业务方法。SyncMixin 统一元数据字段，避免每个模型重复定义。

📦 子系统自包含

file_system 15 个文件自包含，只依赖 3 个第三方包。AI 改子系统时不需要理解全局上下文。

🛡️ 测试即规格

测试 = AI 的安全围栏。每个 Service 方法至少 3 个测试（正常/边界/错误），断言不可变行为不被破坏。

🤝 AGENTS.md 自描述

每个子系统根目录放 AGENTS.md：做什么、对外接口是什么、不能碰什么。AI 读了就知道约束。

⚙

API Layer

4 类端点：REST (/api/v1 CRUD)、Sync (/api/sync push/pull/full)、Export (/api/export jsonl/graph)、MCP (Agent 原生接入)。路由只做校验 + 调 Service + commit。

🔧

Service Layer

9 个核心 Service，全部 flush 不 commit。NoteService 用 Saga 模式桥接 DB 和 file_system。SyncService 内部用 SAVEPOINT 隔离每个事件。

💾

双存储层

pomodoroxii.db 存元数据和热数据（SQLAlchemy + Alembic），file_system 存 .md 正文（FTS5 全文搜索 + 版本历史）。Saga 模式保证跨库原子性。

✅

Outbox 合并矩阵

create+delete→drop both, update+update→keep latest, delete+create→update。客户端不产生重复事件。

🗑

Tombstone 多层防护

删除先建墓碑 → pull 时跳过已删 → 清理 30 天过期墓碑 → revive 防复活检查。四层防护不丢数据。

❓

sync 双路径

note/folder → NoteFileService (Saga 桥接 DB+FS)，其他 12 种实体 → ORM 直处理。adapter 分发，互不影响。

⏱

LWW + version 字段

当前用 updated_at 做 Last-Write-Wins。Phase 3 可选引入 HLC（混合逻辑时钟）替代纯时间戳比较。

🛡️ 可靠性保障

四道防线保证元数据不丢失：同步可靠 + 数据可靠 + 风险缓解

🛡️

同步可靠性

Outbox 合并矩阵防重复 | SAVEPOINT 隔离单事件回滚 | Tombstone 多层防误删复活 | 零点时间戳检测自动修复

💾

数据可靠性

Alembic 迁移替代 _migrate_schema | 双库自动备份(pomodoroxii.db + index.db)每天 03:00 | .md 原子写入临时文件 + os.replace

🚨

风险缓解

Saga 补偿操作跨库失败回滚 | 一致性检查DB ↔ .md 定期扫描 | CascadeService统一级联防孤儿记录
