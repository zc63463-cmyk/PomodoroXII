# PomodoroXII 扩展性 4.5 星提升 — 收尾修复执行计划

## 1. Summary

在 Phase F-1 扩展性 4.5 星提升完成后，审查发现仍存在三类交付阻断点：

1. **测试隔离不足**：`tests/conftest.py` 未清理 `tmp_path/index/`，导致 file_system 测试间脏数据污染，引发 `test_note_ops.py::TestListNotes::test_returns_all_active` 等 CI 失败。
2. **Parity gate 仍依赖硬编码**：`test_parity_stats_mcp.py`、`test_stat_spec.py`、`test_parity_routes.py` 中仍存在硬编码 path / tool 清单，未真正从 FastAPI router 与 MCP server 自省获取实际值。
3. **报告与 CI 元数据过期**：审计报告第 6 节对工作区状态描述错误，`.github/workflows/ci.yml` 中测试数量注释已漂移。

本计划旨在修复上述问题，使全量 pytest 在 CI 中变绿，并消除文档/注释中的易腐数字。

---

## 2. Current State Analysis

### 2.1 测试隔离（`backend/tests/conftest.py`）

- `tmp_path` 被覆盖为 `backend/tests/<test_name>`，每个测试有独立目录。
- `_isolate_env` 已清理 `*.db`、`spaces/`、`notes/`，但未清理 `index/`。
- `test_file_system/conftest.py::fs_instance` 使用 `tmp_path / "index" / "index.db"`。
- 历史测试在 `tests/test_returns_all_active/` 目录留下脏 `index/index.db`，后续运行该测试时 `list_notes()` 读到 16 条历史记录而非当前 2 条。

### 2.2 Parity gate 现状

- `test_stat_spec.py` 硬编码 `expected` REST paths 和 MCP tool 集合。
- `test_parity_stats_mcp.py` 同样硬编码 `/overview` 等 7 个 path 作为反向校验。
- `test_parity_routes.py` 用 `inspect.getsource(build_v1_router)` 字符串计数 `include_router`，并硬编码 16 个 prefix。
- `app/registry/builtin.py` 中没有任何实体设置 `route_enabled` / `route_prefix`，`EntitySpec` 虽已支持这两个字段，但未启用。

### 2.3 CI / 报告

- `.github/workflows/ci.yml` 第 11 行注释仍写 `pytest 524 tests`，第 74 步名称写 `Run pytest (524 tests)`，与当前 536 测试不一致。
- `审计报告/扩展性4.5星提升-修复优化工作报告.md` 第 6 节称"仅 4 个未提交文件"，与当前工作区状态不符；又称"4 个历史失败不影响 CI 判定"，但 CI 实际全量运行 pytest，任何失败都会使 job 变红。

## 3. Proposed Changes

### 3.1 修复测试隔离（`backend/tests/conftest.py`）

**What**：在 `_isolate_env` 中增加对 `tmp_path / "index"` 目录的安全清理。

**Why**：`fs_instance` 在该目录创建 `index.db`，历史残留会导致 list_notes 看到脏数据。

**How**：
- 在 `_isolate_env` 中，删除 `*.db`、 `spaces/`、`notes/` 之后，增加：
  ```python
  index_dir = tmp_path / "index"
  if index_dir.exists():
      shutil.rmtree(index_dir, ignore_errors=True)
  ```
- 仅清理 `tmp_path` 子目录，不会误删仓库外路径。

### 3.2 补充测试保险（`backend/tests/test_file_system/test_note_ops.py`）

**What**：在 `TestListNotes::test_returns_all_active` 中增加显式前置断言或调整 fixture 清理。

**Why**：即使 `_isolate_env` 已修复，也要让该测试自身对"index 为空"敏感，防止未来 fixture 变更再次引入回归。

**How**：
- 在 `test_returns_all_active` 开头先调用 `await fs_instance.list_notes()` 并断言为空；或
- 在 `test_file_system/conftest.py::fs_instance` 的 teardown 中删除 `index/` 目录（在 `yield` 之后执行）。
- 推荐采用后者，因为它在 fixture 层面保证隔离，不改动测试逻辑。

### 3.3 为一级 REST 实体补齐 `route_enabled` / `route_prefix`

**What**：在 `backend/app/registry/builtin.py` 中为具有一级 REST router 的实体设置这两个字段。

**覆盖范围**（用户确认）：

| 实体 | route_prefix |
|------|--------------|
| task | `/tasks` |
| session | `/sessions` |
| note | `/notes` |
| folder | `/folders` |
| quick_note | `/quick-notes` |
| reflection | `/reflections` |
| habit | `/habits` |
| schedule | `/schedules` |
| time_block | `/time-blocks` |
| setting | `/settings` |
| space | `/spaces` |

**不覆盖**（保持 `route_enabled=False`）：
- `auth`、`meta`、`trash`、`stats`、`sync`：非实体 router 或横切能力
- `tombstone`、`sync_outbox`、`sync_audit_log`：sync infra 内部表
- `habit_check_in`、`memo_comment`、`session_quick_note`、`schedule_quick_note`、`task_quick_note`：junction / 无独立一级 router

**Why**：让 `REGISTRY` 成为路由 parity gate 的事实源之一，同时避免让 `EntitySpec` 表达非实体 router 的复杂语义。

**How**：在每个对应 `EntitySpec(...)` 调用中增加 `route_enabled=True, route_prefix="/xxx"`。

### 3.4 重构 `test_parity_routes.py`

**What**：从 `build_v1_router()` 生成的实际 routes 与 `REGISTRY.route_enabled` 做双向静态 parity check。

**Why**：消除对 `inspect.getsource()` 字符串和硬编码 prefix 列表的依赖。

**How**：
- 构造 `router = build_v1_router()`。
- 使用 `router.routes` 提取所有 route path（注意 `APIRoute` 的 `.path` 包含 `/api/v1` prefix 和子 prefix，需要处理成相对 `/api/v1` 的子路径或比较完整路径）。
- 从 `REGISTRY` 收集 `route_enabled=True` 的 `route_prefix`。
- 校验：每个 `route_enabled` 实体的 prefix 都在实际 router 中存在。
- 校验：实际 router 中属于实体 CRUD 的 prefix 都被 `route_enabled` 覆盖（可选，避免反向漏登记）。
- 对非实体 router 使用独立常量列表校验：
  ```python
  EXPECTED_NON_ENTITY_PREFIXES = ["/auth", "/spaces", "/meta", "/trash", "/stats", "/sync"]
  ```

### 3.5 重构 `test_parity_stats_mcp.py` 与 `test_stat_spec.py`

**What**：从 `app.routes.v1.stats.router` 提取实际 REST paths，从 `app.mcp.server` 收集实际 MCP tools，与 `STAT_SPECS` 做双向覆盖。

**Why**：消除硬编码 path / tool 清单，让 parity gate 真正跟随代码变化。

**How**：
- 共享 helper 函数（可放在两个测试文件中都定义，或抽到一个 test helper）：
  ```python
  def _get_stats_rest_paths() -> set[str]:
      from app.routes.v1.stats import router
      return {route.path.replace("/api/v1/stats", "") or "/" for route in router.routes}
  ```
  注意处理 `/api/v1/stats/overview` → `/overview`。
- 从 `STAT_SPECS` 得到 `rest_paths = {s.route_path for s in STAT_SPECS}`。
- 双向 assert：
  - `expected - actual`：STAT_SPECS 中路径未在 router 中实现
  - `actual - expected`：router 中路径未在 STAT_SPECS 中登记
- MCP 工具双向 assert：
  - `expected_mcp = {s.mcp_tool for s in STAT_SPECS if s.mcp_enabled}`
  - `actual_mcp = _get_mcp_tool_names()`（已有实现）
  - 校验 `expected - actual` 和 `actual - expected` 均为空
- `test_stat_spec.py` 中同步采用上述自省方式。

### 3.6 更新 `.github/workflows/ci.yml`

**What**：将易腐测试数量注释改为非易腐表述。

**How**：
- 第 11 行：`pytest 524 tests + ruff lint` → `pytest + ruff lint`
- 第 74 行：`Run pytest (524 tests)` → `Run pytest`
- 同步更新第 5 行描述中的数字（如有）。

### 3.7 更新审计报告

**What**：修正 `审计报告/扩展性4.5星提升-修复优化工作报告.md` 第 6 节。

**How**：
- 删除"仅 4 个未提交文件"表述，改为说明当前工作区存在未提交变更（具体以 `git status` 为准）。
- 修正"4 个历史失败不影响 CI 判定"为：CI 全量运行 pytest，任何失败都会使 job 失败；本次修复目标就是让这 4 个由隔离问题导致的失败变绿。
- 增加一行说明本次收尾修复内容（测试隔离 + parity gate 自省 + CI 注释）。

## 4. Assumptions & Decisions

| 决策 | 说明 |
|------|------|
| 不改 `build_v1_router()` 装配方式 | 保持显式 include_router，避免动态装配带来的导入顺序与可读性风险 |
| `route_enabled` 仅覆盖 11 个一级 REST 实体 | auth/meta/trash/stats/sync 以及 junction/sync_infra 不纳入 |
| 清理 `index/` 仅针对 `tmp_path` 子目录 | 防止误删仓库外路径 |
| 使用 `router.routes` 做自省 | FastAPI `APIRouter.routes` 在装配完成后可用，适合 parity check |
| 不新增测试 helper 模块 | 为减少改动面，helper 逻辑可在两个测试文件中重复或就地定义 |

## 6. Rollback & Safety

- 所有清理逻辑均限定在 `tmp_path` 子目录内，不影响真实 `backend/data/`。
- `builtin.py` 变更仅增加可选字段，不破坏现有 `EntitySpec` 使用方。
- parity test 重构后仍为纯断言，不修改运行时行为。
- 若全量 pytest 仍失败，优先区分：
  - 隔离问题 → 本次范围内继续修复
  - 非隔离问题 → 如与本次改动无关，记录为剩余技术债
