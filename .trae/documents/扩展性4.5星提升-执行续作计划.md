## 假设与决策（继承 v2 计划）

1. **键名兼容**: `EntitySpec` 新增 `sync_entity_type`（camelCase），不填则 fallback 到 `name`
2. **pull_key 显式**: 不依赖复数化规则，所有 14 个 sync 实体显式声明
3. **model_path 动态解析**: 新增 `app/registry/resolve.py` 提供 `resolve_model(spec)` 和 `import_all_models()`
4. **trash 不破坏 FS**: `file_system/engine/trash_ops.py` 独立于 DB trash，本计划不动 FS 层
5. **特殊实体豁免**: Note（FS+DB split）、Folder（cascade）、Relation（junctions）在 parity test 中显式声明例外
6. **脚手架生成显式代码**: 不做运行时黑盒 CRUD，`scaffold_entity.py` 生成可读 Python 文件
7. **Stats 不强行抽象**: Phase 4 用 `StatSpec` 登记 + parity test，不动态生成路由
8. **MCP 测试环境**: 使用 `--capture=no -p no:cacheprovider` 绕过 Trae 沙箱 tempfile 限制
9. **TDD 纪律**: 每个新功能先写**一个**失败测试，观察失败原因正确，再写最小实现；不批量写测试
