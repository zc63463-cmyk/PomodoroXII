## 假设与决策

### 核心决策

1. **键名兼容**: `EntitySpec` 新增 `sync_entity_type` 字段（camelCase），不填则 fallback 到 `name`。所有 14 个 sync 实体显式声明 camelCase 名称，保证协议不变。
2. **pull_key 显式**: 不依赖复数化规则（避免 `memoComment` → `memoComments` 误伤），所有实体显式声明。
3. **model_path 动态解析**: 新增 `app/registry/resolve.py` 提供 `resolve_model(spec)` 和 `import_all_models()`，用 `importlib`。
4. **trash 不破坏 FS**: `file_system/engine/trash_ops.py` 独立于 DB trash，本计划不动 FS 层。
5. **特殊实体豁免**: Note（FS+DB split）、Folder（cascade）、Relation（junctions）在 parity test 中显式声明例外。
6. **脚手架生成显式代码**: 不做运行时黑盒 CRUD，`scaffold_entity.py` 生成可读 Python 文件。
7. **Stats 不强行抽象**: Phase 4 用 `StatSpec` 登记 + parity test，不动态生成路由（保留显式代码）。

### TDD 纪律

- 每个新功能先写**一个**失败测试，观察失败原因正确，再写最小实现
- 不批量写测试再实现（违反 TDD）
- 重构阶段保持测试绿
- 不保留"参考代码"——如果探索性写了实现，TDD 前删除

```python
"""Dynamic model resolution utilities for the registry."""
from __future__ import annotations
import importlib
from app.registry.entities import EntitySpec

def resolve_model(spec: EntitySpec):
    """Import and return the ORM class referenced by spec.model_path."""
    module_path, _, class_name = spec.model_path.rpartition(".")
    module = importlib.import_module(module_path)
    return getattr(module, class_name)

def import_all_models() -> None:
    """Import all ORM models referenced by the registry."""
    from app.registry import REGISTRY
    for spec in REGISTRY.list():
        resolve_model(spec)
```

```python
@dataclass(frozen=True)
class EntitySpec:
    name: str
    model_path: str
    table_name: str
    storage_type: StorageType
    category: EntityCategory
    sync_enabled: bool
    soft_delete: bool
    fields: tuple[FieldSpec, ...]
    primary_key: str = "id"
    description: str = ""
    # 新增扩展字段
    sync_entity_type: str | None = None      # camelCase sync 协议名，None 时 fallback 到 name
    pull_key: str | None = None              # sync pull 响应键名
    route_prefix: str | None = None          # REST 路由前缀，如 "quick-notes"
    service_path: str | None = None          # "app.services.quick_note.QuickNoteService"
    schema_module: str | None = None         # "app.schemas.quick_note"
    schema_prefix: str | None = None         # "QuickNote"
    delete_strategy: str = "hard_tombstone"  # hard_tombstone|soft_delete|cascade_soft_delete|fs_saga|none
    route_enabled: bool = False
    mcp_schema_enabled: bool = True

    @property
    def field_names(self) -> tuple[str, ...]:
        return tuple(f.name for f in self.fields)

    @property
    def effective_sync_entity_type(self) -> str:
        """Return sync_entity_type if set, else name (snake_case fallback)."""
        return self.sync_entity_type or self.name

    @property
    def effective_pull_key(self) -> str:
        """Return pull_key if set, else derive from effective_sync_entity_type + 's'."""
        if self.pull_key:
            return self.pull_key
        return self.effective_sync_entity_type + "s"
```

```python
"""Derive SyncService.ENTITY_REGISTRY from REGISTRY (single source of truth)."""
from __future__ import annotations
from typing import Any
from app.registry import REGISTRY
from app.registry.resolve import resolve_model

def build_sync_registry() -> dict[str, dict[str, Any]]:
    """Build the sync entity registry from REGISTRY.list_sync_enabled().
    
    Returns dict keyed by sync_entity_type (camelCase) with:
        {"model": ORMClass, "pull_key": str, "spec": EntitySpec}
    """
    result = {}
    for spec in REGISTRY.list_sync_enabled():
        model = resolve_model(spec)
        entity_type = spec.effective_sync_entity_type
        pull_key = spec.effective_pull_key
        result[entity_type] = {"model": model, "pull_key": pull_key, "spec": spec}
    return result
```

## 风险与缓解

1. **camelCase 协议破坏**: P2.2 测试 `test_sync_entity_types_match_legacy_registry` 显式锁定 14 个 camelCase 名称，任何漂移立即失败。
2. **ORM 模型动态 import 性能**: `resolve_model` 用 `importlib`，每次调用会查 `sys.modules` 缓存，无性能问题。`build_sync_registry()` 只在模块加载时执行一次。
3. **parity test 误报**: 例外表（P1.8）显式声明允许的偏离，避免 Note.content 等已知特殊情况误报。
4. **Phase 3 脚手架生成质量**: 生成的代码必须通过 `ruff check` + `pytest`，否则脚手架本身失败。
5. **TDD 纪律**: 每个 RED 测试必须观察失败原因正确，再写 GREEN。不批量写测试。
