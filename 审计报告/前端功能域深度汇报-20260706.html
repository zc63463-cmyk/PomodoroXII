八、行动建议

  1. 先 merge Phase D PR — 前端 OpenAPI 以 main HEAD 为准，避免对着 feature 分支/swagger 漂移。
  2. S0 单独立项 — feat/frontend-scaffold：只交付 login、切 space、空 dashboard、Dexie 切换。
  3. S1 Sync 与 S2 Timer 部分并行 — Sync 用 mock UI；Timer 可先 REST 直读后接 offline。
  4. Notes 域单独 PR 链 — list → editor → memo/trash，与后端 Phase D 一一对应。
  5. 每个 Sprint 维护「需求调整表」 — 记录相对 Vue 的增删（多 space、软删 UI、三游标等）。
  6. 默认 Out of Scope — Pomodoroxi-plus 的 Mental Model / Cytoscape 图复盘；Stats AI 洞察（无 API 时）。

建议二期 / 可砍功能

  • Timer 沉浸式模式、白噪音、认知标记 → MVP 后 P1.5
  • Stats AI 洞察卡片 → 需后端 API 或纯本地统计
  • ScheduleView 旧版（398 行）→ 只用 ScheduleCalendarView
  • Reflection 图模式（plus 分支 Cytoscape）→ 非 MVP
